Creating the Global Script

On this page we will create a script that persists between levels and resets.

Creating the Script

Making a global script is very similar to any other, though one key part is that we do not need it to be attached to any node. To create a script not attached to a node, we can right click in our file system, select new, then script.

Go ahead and name the new script Global, then save it. For now we can remove all the code within it other than extends Node. In order to make the script load, however, we need to go to project settings. Select the Autoload tab, then press the file icon next to the path bar. Find your global script, select it, then press Add.

And that is the global script added. In order to prove it, you can run your game, then (while the game is running) look at the scene tree and select remote. You should see your script appear above your world node like so:
